Strategic Deployment Options

Choose the deployment strategy that best leverages your existing infrastructure and operational requirements.

Recommended

Production BIND9 Integration

Perfect for ISPs with existing BIND9 infrastructure. The platform integrates seamlessly with your current DNS server, adding tenant subdomains automatically.

  • ✓ Uses your existing BIND9 server
  • ✓ Automatic zone file management
  • ✓ Enterprise-grade DNS performance
  • ✓ Full integration with existing DNS policies
DNS_STRATEGY=bind9
BIND9_ZONE_DIR=/etc/bind/zones
BIND9_RELOAD_CMD="rndc reload"

# Auto-detected BIND9 ✅
# Creating tenant DNS records...
# Reloading BIND9 configuration...
Enterprise

Router/Firewall DNS

Leverage DNS services built into enterprise routers and firewalls like pfSense, OPNsense, or Cisco equipment. Perfect for smaller ISPs.

  • ✓ Works with pfSense, OPNsense, Mikrotik
  • ✓ No additional servers required
  • ✓ Integrated with firewall rules
  • ✓ API-based management
ROUTER_DNS_ENABLED=true
ROUTER_DNS_API_URL=https://pfsense.local
ROUTER_DNS_API_KEY=your-api-key

# Using router DNS ✅
# Creating DNS entries via API...
# pfSense DNS updated ✅
Development

Docker Container Networking

Perfect for development, testing, or containerized production deployments. Uses Docker's internal DNS with automatic service discovery.

  • ✓ Zero configuration required
  • ✓ Perfect for development teams
  • ✓ Automatic container networking
  • ✓ Works with Docker Compose
DOCKER_DNS_ENABLED=true
DOCKER_DNS_BRIDGE_IP=172.20.0.1

# Using Docker DNS ✅
# Creating internal DNS entries...
# Containers accessible via:
# tenant1.local ✅
Cloud Native

Kubernetes DNS Integration

For cloud-native deployments, integrate with Kubernetes DNS services like CoreDNS for automatic service discovery and DNS management.

  • ✓ Native Kubernetes integration
  • ✓ Horizontal pod autoscaling
  • ✓ Service mesh compatible
  • ✓ Cloud provider agnostic
K8S_DNS_ENABLED=true
K8S_DNS_SERVICE=kube-dns
K8S_NAMESPACE=dotmac-system

# Using CoreDNS ✅
# Creating K8s DNS entries...
# Services available via ingress ✅

Quick Start

Get your ISP management platform running in minutes

1

Clone and Configure

git clone https://github.com/yourusername/dotmac-platform.git
cd dotmac-platform
cp .env.example .env

# Edit configuration
nano .env
# Set BASE_DOMAIN=yourdomain.com
# Set LOAD_BALANCER_IP=your.server.ip
2

Deploy Platform

# Platform auto-detects your infrastructure
./scripts/deploy-production.sh

# System will detect and use:
# ✅ BIND9 (if available)
# ✅ Docker DNS (if containerized)  
# ✅ Router DNS (if configured)
# ✅ Hosts file (development fallback)
3

Create Your First Tenant

# Deploy a tenant ISP
./scripts/deploy-tenant.sh acme-isp

# Automatic setup:
# ✅ DNS records: acme-isp.yourdomain.com
# ✅ Database: isolated tenant schema  
# ✅ Portals: customer.acme-isp.yourdomain.com
# ✅ API: api.acme-isp.yourdomain.com
4

Access Management Portals

  • Admin Portal: https://acme-isp.yourdomain.com
  • Customer Portal: https://customer.acme-isp.yourdomain.com
  • API Documentation: https://api.acme-isp.yourdomain.com/docs
  • Monitoring: https://monitoring.yourdomain.com
